Standards in Website Design

When it comes to Website design there are a lot of standards that you are expected to follow. Every language has their own 'good practice' and in general there are some dos and don'ts of websites altogether.

The World Wide Web Consortium (Often abbreviated to W3C) is an international organization that is in charge of producing technical standards for the web. The community has a very rigorous series of steps that all propositions go through, and it can often take years before the suggestion is eventually accepted and established as a standard. The W3C also works to teach individuals, and is the creator of the w3schools website which is probably the most used resource by aspiring web developers.

Different browsers display HTML elements differently and not all browsers have the same HTML handling capability. It is obvious that we cannot go back to Internet Explorer 1 and use the same HTML as a website from Internet Explorer 11 and expect it to work, but surprisingly we sometimes cannot use a website that functions perfectly well in one modern browser and expect it to work in another. Each web browser updates to include new HTML and CSS features in their own ways, while Google just has frequent updates to the whole browser Internet Explorer would instead release entire new versions of itself with different features included. For people who use older browsers, this can be an issue. If someone is using a browser that is too old (for example, Internet Explorer 8) then it is quite likely that a feature you have does not load for them or is not shown how you want it to be. This can cause a lot of issues depending on the importance of the feature and it is one of many reasons why some websites do not use all of the newest features if they are trying to pander somewhat more towards individuals using outdated software. A useful JavaScript library is Modernizr, which contains many helpful scripts that allow you to check if a feature works on a user's browser and if it doesn't then you can set a backup feature instead.

You might have noticed that nearly every website you connect to will start the URL with 'https://' or 'http://'. HTTP stands for the Hyptertext Transfer Protocol and is the set of instructions a computer follows that specify how to transfer and process data through the WWW. HTTPS is a version of HTTP which has increased security by using SSL encryption (thats where the S comes from) and makes the data sent to the website less likely to be stolen or used maliciously. HTTP and HTTPS are the standards for communication over the WWW, with HTTPS now being the more used protocol since 2018.

Some websites, like this one, try to adhere to specific guidelines that allow the website to remain 'accessible'. One major design accessibility feature is to do with the use of colour in websites. It is stated that approximately 5% of people suffer from some form of colour blindness, and people with poor eyesight can struggle to read information when the colours used aren't very distinguishable. W3C's Web Content Accessibility Guidelines suggest that the contrast ratio between the foreground and background colours should be minimum of 4.5:1 for text on a website. This isn't a perfect value, and people with extreme visual impairedness may still struggle at this point, but this is a good minimum to have that ensures nearly all users will be able to read text on the website.

When writing HTML you should always do your best to keep both the website and the code 'clean looking', that is to say that it should not be messy to read or navigate. There are some principles of best practice you should try to follow when writing HTML to ensure your code is good:

  • Pay Attention to what you write and how much effort you put in
  • Indent your code properly to make it easy to read and modify
  • Keep your styling in the CSS file, and keep your scripting in a JS file
  • While divs are useful to keep structure, don't overuse them when they aren't needed. Avoid 'divitis'
  • Use one tag per line
  • Class tags and IDs should have meaningful names
